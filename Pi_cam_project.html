Pi Video Streaming for Distributed Processing

For this project, I aimed to enhance my skills in Raspberry Pi development, programming, and general networking by creating two video streaming systems. The objective was to capture video of a moving red dot on one device, send the video to another device for processing to track the dot's movement between frames, and finally send the movement infomation back to be displayed. The configurations included streaming and processing video between two Raspberry Pis, and between a Raspberry Pi and a PC.

Using a Linux-based operating system on the Raspberry Pi, I programmed the systems using Python in the MicroPython interpreter, Thonny. Key libraries employed were OpenCV for video capture and processing, the socket library for network communication via UDP, and the pickle module for serializing data for transmission.

Upon creation of the system, I analysed the different qualities of each configuration to compare the quality of different transmission methods. This was done using wireshark which is a network protocol analyser that captures packets from a network connection.
